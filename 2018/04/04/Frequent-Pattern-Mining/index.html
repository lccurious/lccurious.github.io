频繁模式挖掘

发表于 2018-04-04 更新于 2020-10-01 分类于 机器学习 阅读次数： 3.3k 3 分钟

在很多时候我们会关注两个或多个对象同时出现的频率，这些相关性往往意味着某种行为特征。例如我们可以根据网站的日志找到网站中某些的网页经常被用户访问到，这些频繁性提供了用户浏览习惯的线索，有助于提高浏览的体验。其中非常典型的应用就是购物篮分析（market basket analysis）可以通过用户经常购买的物品集合找到人们经常一起买的物品集合。之后可以提取出这些物品的关联规则（association rule）比如可以发现啤酒和尿布经常会被人组合购买，从而将这些物品放在邻近的地方销售会有更好的效果。

项繁集和事务标识符集

令I={x1,x2,…,xm}\mathcal{I}=\{x_1, x_2, \dots, x_m\}I={x1​,x2​,…,xm​}为一组称为项(item)的元素集合例如超市中售卖的所有产品的集合。集合X⊆IX\subseteq \mathcal{I}X⊆I称为项集。一个基数为k或者大小为k的项集称为k项集，用I(k)\mathcal{I}^{(k)}I(k)表示所有k项集。

令T={t1,t2,…,tn}\mathcal{T}=\{t_1, t_2, \dots, t_n\}T={t1​,t2​,…,tn​}为另一个所谓的事务标识符(tid)构成的集合。事务（transaction）是一个形如⟨t,X⟩\langle t, X\rangle⟨t,X⟩的元组，其中t∈Tt\in \mathcal{T}t∈T是一个独一无二的标识符，XXX是一个项集。事务的集合T\mathcal{T}T可以代表超市中的所有客户等。方便起见可以用ttt来表示一个⟨t,X⟩\langle t, X\rangle⟨t,X⟩。

一个二元数据库D\mathbf{D}D表示了事务标识符集和项集之间的二元关系。

数据集D\mathbf{D}D的一个项集XXX的支撑（support），表示为sup⁡(X,D)\sup(X,\mathbf{D})sup(X,D)，即D\mathbf{D}D中所有包含XXX的事务的数目：

根据规则支撑和置信度的定义，可以观察到，为了生成频繁且高置信度的关联规则，首先要枚举所有的频繁项集及其支撑值。

暴力枚举

首先要做候选生成，因为一个集合I\mathcal{I}I有2∣I∣2^{\mid \mathcal{I} \mid}2∣I∣个子集也就有2∣I∣2^{\mid \mathcal{I} \mid}2∣I∣个可能的频繁项集。所以可以用前缀树通过BFS或者DFS来枚举项集。

然后要完成支撑计算，这一步计算每一个候选模式XXX的支撑，并且判断它是否为频繁的。

但是这种方式的计算复杂度太大，支撑的计算在最坏的情况下需要O(∣I∣⋅∣D∣)O(\mid \mathcal{I} \mid \cdot\mid\mathbf{D}\mid)O(∣I∣⋅∣D∣)的时间。由于一共有O(2∣I∣)O(2^{\mid \mathcal{I} \mid})O(2∣I∣)个可能的候选，暴力枚举的复杂度为O(∣I∣⋅∣D∣⋅2∣I∣)O(\mid \mathcal{I} \mid \cdot\mid\mathbf{D}\mid\cdot 2^{\mid \mathcal{I} \mid})O(∣I∣⋅∣D∣⋅2∣I∣)。这种复杂度在现实中是不可接受的。

逐层的方法：Apriori算法

根据两个依据进行剪枝：

  1. 若XXX是频繁的，则其任意子集Y⊆XY\subseteq XY⊆X也是频繁的
  2. 若XXX不是频繁的，则任意超集Y⊇XY\supseteq XY⊇X都不是频繁的
